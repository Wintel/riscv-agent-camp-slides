# 第三讲 Agent 具体应用 · PPT 大纲

## 定位

- 面向对象：零 Linux 基础的高中生。
- 承接关系：前一讲已经讲过 Agent、LLM、工具调用、规划、记忆，以及 OpenClaw 的基本上手。
- 本讲目标：让学生从“知道智能体是什么”，走到“知道什么任务适合交给它、怎样把任务说清楚、怎样验收结果”。
- 表达方式：继续沿用前两份讲义的中文、案例驱动、少抽象多场景的风格，尽量把例子放在 Titan 板子和日常开发里。

## 建议页数

- 建议 28 到 30 页。
- 前 4 页做 recap。
- 中间主体讲“应用场景 + 设计方法 + 真实案例”。
- 最后 5 到 6 页讲动手体验、实践任务和收尾。

## 本讲主线

一句话主线：

> 智能体不是“自动替你做一切”的魔法，而是一个会调用工具的搭子。真正重要的是：选对任务、给足上下文、设好边界、自己验收。

## 逐页大纲

1. 封面：`智能体介绍 III · Agent 具体应用`
2. Recap：昨天学了什么
   智能体 = 大模型 + 工具 + 想做看循环；OpenClaw 负责把你的话变成对 Titan 的操作。
3. Recap：昨天带走的七般武艺
   读文件、改代码、跑命令、修报错、查资料、讲解代码、辅助完成小任务。
4. Recap：昨天的使用心法
   4D：描述、委派、辨别、尽责；再提醒三个坑：胡说、兔子洞、拐杖。
5. 今天要解决的问题
   什么任务适合交给 Agent？怎样设计任务？怎样把结果真的落地？
6. 应用全景
   学习助手、开发搭子、实验助手、产品合伙人四类典型角色。
7. 任务筛选
   适合：重复、步骤明确、结果可验证的任务。
   不适合：高风险、目标模糊、无法验证、涉及关键秘密的任务。
8. 案例 1：板子体检员
   刚拿到一块 Titan，让 Agent 帮你快速检查系统、磁盘、内存、网络信息。
9. 案例 1 拆解
   命令 -> 输出 -> 整理成报告 -> 给出下一步建议。
10. 案例 2：修 bug 搭子
    把报错原文贴给 Agent，让它先定位问题，再提出最小修改。
11. 案例 2 拆解
    定位 -> 修改 -> 运行验证命令 -> 复盘为什么错。
12. 案例 3：小产品合伙人
    让 Agent 给 Titan 做一个小网页或小工具，而不只是停留在命令行。
13. 案例 3 拆解
    需求 -> 页面原型 -> 本地运行 -> 再提改进意见。
14. 更多还能做什么
    资料整理、实验记录、命令解释、日志归纳、报告润色。
15. 设计方法总览
    五件事：输出物、上下文、工具、约束、验证。
16. 先写清输出物
    “帮我做个页面”太模糊；“做一个显示主机名/时间/内存/磁盘的单页网页”更可执行。
17. 再给上下文
    当前目录、已有文件、系统环境、目标路径、不能碰的内容，都要说清楚。
18. 工具与权限
    默认先做低风险操作；写文件前说明；安装依赖、联网、删除内容前先请示。
19. 验证与回滚点
    不要只听“我做完了”，要看命令输出、看页面结果、看文件改动、留回退点。
20. 人机分工
    AI 负责搜集、试错、搬运、生成初稿；人负责目标、判断、取舍和最终拍板。
21. 可复用 Prompt 模板
    任务 + 背景 + 限制 + 验收标准 + 汇报方式。
22. 今天的实战题
    做一个 `Titan 状态看板`：展示主机名、时间、CPU 架构、内存、磁盘占用。
23. 第一步：把任务书交给智能体
    给出范围：不用安装依赖、只在当前目录工作、完成后本地预览。
24. 第二步：看它计划和执行
    看它先查信息、再生成页面、再启动本地服务。
25. 第三步：自己验收
    检查网页是否能打开，信息是否正确，目录是否整洁，命令是否可复现。
26. 第四步：继续迭代
    追加需求：移动端适配、刷新按钮、颜色警示、说明文字。
27. 下午实践菜单
    任务 A：状态看板。
    任务 B：错误猎人。
    任务 C：命令讲解器。
28. 常见卡点
    任务太大、上下文太少、验收缺失、绕圈不止、把“建议”当“事实”。
29. 回顾
    从“会用智能体”进到“会设计智能体任务”。
30. 谢谢 + 资源 + 挑战题
    留下课后资源，并给学生一个可以自己回去继续玩的延伸任务。

## 讲法提示

- recap 控制在 3 到 4 页，不重复昨天的完整内容。
- 每个案例都要有三个层次：
  场景句 -> 智能体做了什么 -> 你最后检查什么。
- “设计方法”部分不要讲成理论课，要一直拿前面的案例反复对应。
- “动手体验”部分尽量给出学生下午就能直接照着做的任务书。
- 收尾明确告诉学生：智能体最有价值的，不是一次性生成很多字，而是帮你把真实任务推进下去。
