03 先认识"大脑"

大模型（LLM）
是什么？

用海量文本（几乎整个互联网）训练出来的"语言大脑"。核心本领简单得惊人：看着前文，猜下一个字。

  • 📚
    读得多像读遍了图书馆——你起个头，它能自然接下去
  • ✨
    练大了会"涌现"大到一定程度，竟然会翻译、推理、写代码
  • ⌨️
    对今天最重要的一点它读过全世界的代码和命令行——是个"老程序员"
今天天气真
大模型猜下一个字（各有概率）：
好70%
热20%
不8%

一个字一个字接下去，就能写出整段话、整个程序。你听说过的 ChatGPT、DeepSeek，底层都是它。

06 运转方式

它怎么自己干活：想 → 做 → 看，转圈圈

🛠️

👀

↺

走一圈："帮我看看这块板子装没装 Python" → 想：查版本该跑哪条命令？ → 做：python3 --version → 看：输出 Python 3.11.2 ✅ → 回来汇报。没办成？再转一圈，直到搞定或来问你。

07 一个关键设定

它的"记性"是一张
有限的书桌

  • 1
    大模型本身没有记性每答一句，都要把整段对话从头再读一遍
  • 2
    书桌大小有限（上下文窗口）聊得越久，桌上的纸越堆越厚
  • 3
    桌子太乱，它会"顾此失彼"开头交代的要求，聊到后面就忘了
记住这个比喻——后面"用得好"的技巧，一半都是在帮它收拾桌子。
第1问你：写个脚本
第2问你：写个脚本它：写好了A你：改成5秒
第3问你：写个脚本它：写好了A你：改成5秒它：改好了B你：加上时间

每次提问，整摞纸都要重新塞给大模型读一遍

🪑 书桌快满时它会整理归档：把前文压缩成一页摘要，细节就丢了

14 武艺一 · 最高频

  • 🥉

  • 🥈

18 功课一 · 描述

写一份好的任务说明书

❌ "帮我写个温度程序"

它只能瞎猜：什么温度？多久测一次？存哪里？
猜错 → 返工 → 你还嫌它笨。

✅ 目标 + 材料 + 验收标准

"写 Python 脚本 cpu_temp.py：
① 每 5 秒读一次 CPU 温度
② 追加写入 temp.log，每行"时间 温度"
③ 验收：跑 1 分钟，文件里正好 12 行"

目标：做什么 材料：报错原文 / 相关文件 / 背景 验收：怎么算做好了
拿不准就加一句："动手前，先把你的计划列给我看。"——纠计划一句话，纠代码半小时。

🔁 派活的最高技巧：给它"反馈闭环"

好任务自带它能自己跑的检查——它错了能自己发现、自己改，不用你盯。

python3 cpu_temp.py --seconds 60 ✓ 12 行，通过

跑偏了？随时打断，补一句话把它拉回来——别等它跑完。
