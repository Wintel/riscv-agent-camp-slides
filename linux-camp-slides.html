14 找到它 · 试探它

板子的 IP 是多少？

  • A
    在板子上直接查通过串口登录后运行 ip addr，找到 192.168 开头的那个
  • B
    看路由器后台路由器管理页里，会列出所有连接设备的 IP
  • C
    用 ping 试试通不通能收到回复，就说明两台机器网络可达
在你的电脑上
$ ping 192.168.1.50 来自 192.168.1.50 的回复: 时间=0.8ms 来自 192.168.1.50 的回复: 时间=0.7ms 来自 192.168.1.50 的回复: 时间=0.9ms # 有回复 = 网络通了 ✓ 按 Ctrl+C 停止 $
ping 像在喊"喂，在吗？"，对方回一声"在！"，还告诉你来回花了多久（毫秒）。

16 今天的主角

SSH = Secure Shell

一句话：安全地、从你的电脑，去操作另一台电脑的命令行。板子没接屏幕、可能在别的房间甚至云端 —— SSH 让你像坐在它面前一样敲命令。

💻
你的电脑
SSH 客户端
🔒 加密隧道
🖥️
Titan 板子
SSH 服务器
🔒 加密

中途被人偷看，也只看到乱码

🌍 远程

板子在哪都能连，只要网络可达

⌨️ 就是命令行

连上后，前面学的命令全都能用

SSH 连接的四步（简化版）

1

2
建立加密隧道

双方协商密钥，之后通信全程加密

18 ★ 核心实操

你的第一次 SSH 连接

在你自己的电脑上
$ ssh student@192.168.1.50 The authenticity of host ... can't be established. ED25519 fingerprint is SHA256:aBcD... Continue? yes ⏎ student@192.168.1.50's password: •••••• Welcome to Ubuntu (RISC-V) 🎉 student@milkv:~$
格式：ssh 用户名@板子IP

就这一条命令，是今天最需要记住的。

👀
第一次会问 fingerprint，确认后输 yes 回车。之后就不再问了。
🙈
输密码时屏幕不显示任何字符，这是正常的安全设计 —— 别以为键盘坏了，照打回车即可。
看到提示符从你电脑的名字，变成了 student@milkv —— 恭喜，你已经在板子里面了！🎉
19 进阶 · 可选

免密码登录：
SSH 密钥

每次输密码太麻烦，也不够安全。更好的方式是用一对"钥匙"。

  • 1
    ssh-keygen生成一对钥匙：私钥（留在自己电脑，绝不外传）+ 公钥
  • 2
    ssh-copy-id student@…把公钥"锁孔"装到板子上
  • 3
    以后 ssh 直接进钥匙自动对上锁，免密码、更安全
🔑
私钥 Private Key

只存在你的电脑里，像家门钥匙，绝不给别人

配对验证 🔒 ⇄ 🔓
🔓
公钥 Public Key

放到板子上，像装好的锁，公开也没关系

22 卡住了别慌

连不上？对照排查

ping 不通 / No route to host

👉 多半没在同一局域网。检查网线、Wi-Fi，确认 IP 前几段一致。

Connection refused

👉 IP 通但 22 号门没开：板子还没启动好，或 SSH 服务没运行。

Permission denied

👉 用户名或密码错了。注意密码区分大小写，重新输一遍。

连着连着断了 / IP 变了

👉 路由器可能重新分配了 IP，重新 ping 查一次最新地址。

📋 终极大招：把完整的报错信息复制给大模型，描述你做了什么，它几乎总能帮你定位问题。
