12 东西都放哪儿

Linux 的几个"标准房间"

树根 / 下这几个文件夹，几乎每台 Linux 都有，各管一摊。认个脸熟就好。

/home

每个用户的"家"，你的文件都在 /home/你

/etc

系统的"设置面板"：各种配置文件

/bin

命令们的家：ls、cat 这些程序本体

/tmp

临时文件，重启可能就清空了

/var

经常变动的东西，比如日志 log

/dev

硬件也是"文件"：硬盘、串口都在这

Linux 一大哲学：一切皆文件 —— 连键盘、硬盘、串口，都被当成 /dev 下的文件来读写。

14 一次搞定一批

Linux 里普通用户不能乱动系统文件（这是一种保护）。要装软件、改系统设置时，在命令前加 sudo 就行。

32 进阶 · 可选

免密码登录：
SSH 密钥

每次输密码太麻烦，也不够安全。更好的方式是用一对"钥匙"。

  • 1
    ssh-keygen生成一对钥匙：私钥（留在自己电脑，绝不外传）+ 公钥
  • 2
    ssh-copy-id student@…把公钥"锁孔"装到板子上
  • 3
    以后 ssh 直接进钥匙自动对上锁，免密码、更安全
🔑
私钥 Private Key

只存在你的电脑里，像家门钥匙，绝不给别人

配对验证 🔒 ⇄ 🔓
🔓
公钥 Public Key

放到板子上，像装好的锁，公开也没关系

39 卡住了别慌

连不上？对照排查

ping 不通 / No route to host

👉 多半没在同一局域网。检查网线、Wi-Fi，确认 IP 前几段一致。

Connection refused

👉 IP 通但 22 号门没开：板子还没启动好，或 SSH 服务没运行。

Permission denied

👉 用户名或密码错了。注意密码区分大小写，重新输一遍。

连着连着断了 / IP 变了

👉 路由器可能重新分配了 IP，重新 ping 查一次最新地址。

📋 终极大招：把完整的报错信息复制给大模型，描述你做了什么，它几乎总能帮你定位问题。
