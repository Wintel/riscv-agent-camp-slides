04 第一站 · 先认零件

先拆开看：一台电脑里有什么

不管是你的笔记本、你的手机，还是等下要发到你手上的这块板子，里面永远是这四样东西在配合。

🧠

CPU 处理器 —— 大脑

负责算。Titan 上是 UR-DP1000，8 个核，等下细讲。

📖

内存 DDR4 —— 书桌

正在用的东西摊在桌上，拿得飞快，但一断电就全没了。

🗄️

硬盘 M.2 SSD —— 书柜

关机也留着。系统、你写的代码，都住在这儿。

🌐

网口 —— 门

和外面世界说话的唯一通道。第 2 天你就是从这道门进去的。

🤔 20 秒抢答：CPU、内存、硬盘，谁负责"临时记事"、一断电就忘光？　答案：内存——那张"书桌"；CPU 只管算，硬盘关机也记得。

这四样谁都有。Titan 特别的地方只有一个：它的大脑，用的是 RISC-V。

10 第二站 · 硬件层

CPU 是怎么
"听懂"命令的？

芯片其实只认一小套最基础的动作：加一下、把数据搬进来、再存回去…… 这套"动作清单"就叫 指令集架构（ISA）。

  • 🍽️
    像餐厅的一份"菜单"厨房（CPU）只会做菜单上的菜；点菜（写程序）也只能照着点
  • 🤝
    软件和硬件的"约定"大家都照这套语言来，芯片就能跑对应的程序
CPU 认得的"基础动作"
add a0, a1, a2 # a0 = a1 + a2（加法） ld a0, 0(a1) # 从内存读一个数进来 sd a0, 0(a1) # 把一个数写回内存 beq a0, a1, L # 两个数相等就跳转

你写的每一行代码，最后都被翻译成这样一条条"基础动作"。

11 第二站 · 硬件层

为什么手机上的 App，装不到电脑上？

同样一句"把两个数加起来"，翻译成机器听得懂的话，不同架构说的是不同方言：

编译到 x86-64（你的笔记本）
// C 代码：int add(int a,int b){ return a+b; } lea eax, [rdi+rsi] ret
编译到 RISC-V（我们的 Titan）
// 同一份 C 代码 add a0, a0, a1 ret
谁多一条谁少一条不重要，重要的是：它们互相不认识。
所以软件必须"为这个架构编译过"才能跑——这就是为什么大家老在说 RISC-V 的"生态"。
12 第二站 · 硬件层

RISC 是什么意思？——菜单短，但够用

CISC · 复杂指令集（x86）

菜单很长，还有不少"一道菜顶三道"的复杂大菜。
点菜方便，但厨房（芯片）要能做几千道菜，设计非常复杂、耗电也大。

RISC · 精简指令集（RISC-V）

菜单短而整齐，复杂的菜用几道简单的菜拼出来。
厨房小、好造、省电，还容易验证有没有做错。

那名字里的 V 呢？它是罗马数字 5——伯克利做的第五代 RISC 设计。

今天
上千家企业与机构加入小到耳机里的芯片，大到数据中心的服务器 CPU，都能看到它。
💡
一个暑假开始的项目，十几年后成了全球标准。
而你们，现在也正在过一个暑期营。

15 第二站 · 硬件层

为什么说 RISC-V 是乐高？

别家的指令集是"整套买"。RISC-V 是一小块基础积木，再按需要往上拼扩展——做耳机就拼小一点，做服务器就拼大一点。

RV64I 基础 · 整数运算 M 乘除 A 原子操作 F / D 浮点 C 压缩指令 B 位操作 H 虚拟化 V 向量 · 跑 AI 常用
把最常用的几块打包，就简称 G。所以规格表里那串"天书"，其实就一句话——
RV64GCBHX ＝ 64 位 + 常用功能包 + 可自由扩展。
想拆开看：G 常用包 · C 压缩 · B 位操作 · H 虚拟化（把一台机器当好几台用）· X 厂商自定义——记不住没关系。

这正是我们板子上 UR-DP1000 的规格——现在你已经能读懂它了。
不过自由也有代价：拼法太多，软件就不知道该按哪套写。这个麻烦怎么解决，两页之后见分晓。

26 第四站 · 智能层

大模型（LLM）
是什么？

用海量文本（几乎整个互联网）训练出来的超大"语言大脑"。它的核心本领简单得惊人：看着前文，猜下一个字。

  • 📚
    读得多像读遍了图书馆，你起个头，它能自然地接下去
  • ✨
    练大了会"涌现"练到足够大，竟然会翻译、写代码、推理、答题
今天天气真
大模型猜下一个字（各有概率）：
好70%
热20%
不8%

一个字一个字这样接下去，就写出了整段话。

27 第四站 · 智能层

它是怎么"学会"的？

大模型的一生，其实只分两种时刻：读书和答题。

训练 · 读书
📚

把海量文本喂给它，让它反复练"猜下一个字"。
要用成千上万张 GPU 卡、连着跑好几个月、烧掉数亿元。一辈子只做有限的几次。

推理 · 答题
⚡

你问一句，它算一遍，给你答案。
几秒钟的事。全世界每天要发生几十亿次——你用的每一次都是这个。

关键推论：它的知识在训练那一刻就定格了。
这就是为什么它不知道今天的新闻——也是下一页要说的"天花板"。
28 第四站 · 智能层

它的书桌
有多大？

模型读东西不是一个字一个字看，而是按 token（词块）看。中文里，一个汉字大约是 1～2 个 token。

一次对话里，能同时摊在桌上的 token 总量，叫 上下文窗口。
桌子摊满了，早先的内容就得先收起来——它就"忘"了。
📄
放什么上桌，是你的活

同样一个问题，把报错原文、文件内容一起给它，和只说"报错了"，结果天差地别。

🧹
任务太长要学会收桌子

阶段性让它总结一下"我们做到哪了"，比让它硬扛一大堆旧内容有用。

🎯
这是第 3、4 讲的重点

"怎么把话说清楚"，比"知道它是怎么造的"实用得多。

29 第四站 · 智能层

但大模型有个"天花板"

它很会"说"，可它只是输出文字——不知道现在几点、算不准复杂的账、连不上网，也没法真正动你的电脑。

🕒

不知道"此刻"

训练之后发生的事、现在的天气，它都不知道

🔌

没有"手脚"

不能自己查资料、跑程序、控制硬件

🎯

只答一句

你问一句它答一句，不会自己一步步把事办完

要让这个"聪明的脑子"真正干活，得给它装上手脚和工具——这就是 智能体。👉
30 第四站 · 智能层

智能体 = 大模型 + 手脚和工具

只有大模型
🧠

一个"超级大脑"
你问一句，它答一句

智能体 Agent
🧠 ＋ 🛠️ ＋ 📝

大脑 + 会用工具的手 + 记得住事的笔记本
给它一个目标，它自己拆解、动手、一步步尝试（也会出错，要你把关）

打个比方：大模型是"聪明的脑子"，智能体是"聪明的脑子 + 会用工具的双手 + 记事的笔记本"。
在我们的课里，这双"手"在 Titan 板子上敲命令，那颗"大脑"（大模型）常常在云端——并不是塞在板子里跑的。
31 第四站 · 智能层

智能体怎么自己干活？

它把一个大目标，拆成"想一步、做一步、看一步"的小循环，转着转着就把事办成了。

🧠
思考下一步该做什么
→
🛠️
行动调用工具 / 写代码 / 查资料
→
👀
观察看看结果对不对
↺
例："帮我在板子上写个程序，读温度并存进文件" → 想：先写代码 → 写好并运行 → 看到报错 → 改一改再跑 → ✅ 跑通。没做完，就再转一圈。
32 第四站 · 智能层

一个回合，
长这样

右边是一次真实的派活。你只说了第一句，剩下的都是它自己转圈转出来的。

注意中间那一步：它犯了错，然后自己看见、自己改掉。
这正是"智能体"和"聊天机器人"的分界线。
🤔 判断：它报告 ✅ 已写入 52.3°C，你敢直接信、直接交作业吗？
先别急着交——AI 报"成功"不等于真的对，得你自己核一眼。
agent @ titan

33 第四站 · 智能层

今天的代码，是这样写出来的

2025.02
AI 科学家 Andrej Karpathy 随手造了个词：vibe coding。 意思是：不再一行行敲代码，而是用大白话告诉 AI 你想要什么，让它去写。
2025.11
柯林斯词典把它选为年度词汇。 一个程序员圈子里的玩笑话，九个月就进了词典——因为真的太多人在这么干了。
2026.04
谷歌 CEO：公司 75% 的新代码由 AI 生成，工程师审核后合入。 2024 年秋这个数字是 25%，2025 年秋约一半。微软 2025 年也公布过约 30%。
🔍
回头看这三条，都跟着同一个小尾巴："由工程师审核"。
AI 负责敲，人负责看懂、判断、拍板。这四天真正要练的，是后面那半句。
