02 第一站 · 系统层

先说 Linux：
这台机器的操作系统

芯片是硬件，还得有个操作系统来指挥它、把程序跑起来。我们这台 RISC-V 电脑用的，就是 Linux。

至于怎么用命令行操作它、怎么用 SSH 远程连上板子——那是 Linux 实操课的动手内容；这节课先认识它就够了。
🆓
开源、免费

代码人人可看、可改、可免费用——和等下的 RISC-V 是同一种精神。

🌍
无处不在

安卓手机、服务器、超算、跑 AI 的机器……几乎都在用 Linux。

🖥️
我们的 Titan 板子

装的正是 Ubuntu / Debian——都是 Linux。

03 第二站 · 硬件层

CPU 是怎么
"听懂"命令的？

芯片其实只认一小套最基础的动作：加一下、把数据搬进来、再存回去…… 这套"动作清单"就叫 指令集架构（ISA）。

  • 🍽️
    像餐厅的一份"菜单"厨房（CPU）只会做菜单上的菜；点菜（写程序）也只能照着点
  • 🤝
    软件和硬件的"约定"大家都照这套语言来，芯片就能跑对应的程序
CPU 认得的"基础动作"
add a0, a1, a2 # a0 = a1 + a2（加法） ld a0, 0(a1) # 从内存读一个数进来 sd a0, 0(a1) # 把一个数写回内存 beq a0, a1, L # 两个数相等就跳转

你写的每一行代码，最后都被翻译成这样一条条"基础动作"。

04 第二站 · 硬件层

RISC-V：一套开放、免费的指令集

过去主流指令集不是 Intel 的 x86，就是 ARM——想造芯片，得付高昂授权费、还处处受限。RISC-V 反过来：开源、免费、谁都能用。

💻

x86

Intel / AMD 主导，几乎不对外授权
电脑 · 服务器

📱

ARM

要付授权费、按规矩使用
手机 · 平板

⭐

RISC-V

开放免费、可自由扩展
从单片机到服务器 · 我们的板子

🧩 精简：指令少而规整，好学好造 🔧 可定制：想加功能自己扩展 ⚖️ 中立开放：由国际基金会维护，不被一家公司说了算
05 第二站 · 硬件层

为什么大家都在押注 RISC-V？

🌍

正在成为潮流

从耳机、单片机到数据中心，越来越多芯片改用 RISC-V。

🇨🇳

自主可控

不必依赖国外授权，是国产芯片的重要机会——我们的 DP1000 就是国产 RISC-V。

🧩

生态在长大

编译器、Linux、各种 AI 框架都陆续支持 RISC-V。

🎓

特别适合学习

开放、资料公开，正好拿来教学和动手研究——就像现在。

一句话：现在学 RISC-V，就像当年早早学了 Linux——站上了一条正在长大的赛道。

08 第四站 · 智能层

大模型（LLM）
是什么？

用海量文本（几乎整个互联网）训练出来的超大"语言大脑"。它的核心本领简单得惊人：看着前文，猜下一个字。

  • 📚
    读得多像读遍了图书馆，你起个头，它能自然地接下去
  • ✨
    练大了会"涌现"练到足够大，竟然会翻译、写代码、推理、答题
今天天气真
大模型猜下一个字（各有概率）：
好70%
热20%
不8%

一个字一个字这样接下去，就写出了整段话。

09 第四站 · 智能层

但大模型有个"天花板"

它很会"说"，可它只是输出文字——不知道现在几点、算不准复杂的账、连不上网，也没法真正动你的电脑。

🕒

不知道"此刻"

训练之后发生的事、现在的天气，它都不知道

🔌

没有"手脚"

不能自己查资料、跑程序、控制硬件

🎯

只答一句

你问一句它答一句，不会自己一步步把事办完

要让这个"聪明的脑子"真正干活，得给它装上手脚和工具——这就是 智能体。👉
10 第四站 · 智能层

只有大模型
🧠

一个"超级大脑"
你问一句，它答一句

智能体 Agent
🧠 ＋ 🛠️ ＋ 📝

大脑 + 会用工具的手 + 记得住事的笔记本

11 第四站 · 智能层

智能体怎么自己干活？

它把一个大目标，拆成"想一步、做一步、看一步"的小循环，转着转着就把事办成了。

🧠
思考下一步该做什么
→
🛠️
行动调用工具 / 写代码 / 查资料
→
👀
观察看看结果对不对
↺
例："帮我在板子上写个程序，读温度并存进文件" → 想：先写代码 → 写好并运行 → 看到报错 → 改一改再跑 → ✅ 跑通。没做完，就再转一圈。
